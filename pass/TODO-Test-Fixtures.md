## How to Generate Test Keys

```bash
cd pass/testdata/gpg
./generate_test_keys.sh
```

This will create:
- `test-gnupg-home/` directory with test keys
- `key-ids.txt` with the generated key IDs

Then update `fixtures.go` with the actual key IDs.

## Notes

1. The test keys are generated with a known passphrase (`test-passphrase-123`) which is safe to commit because it's only used with test keys.

2. The `test-gnupg-home/` directory should be committed to the repository so tests can use pre-generated keys.

3. Tests should use `SetupTestEnv()` or `SetupTestEnvWithGPGKeys()` to set up a clean test environment.

4. All test data is automatically cleaned up after tests complete.

5. The implementation follows the repository's AGENTS.md guidelines for spec-driven development.
